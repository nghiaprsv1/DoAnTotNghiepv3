Cấu trúc thư mục & file của TripMate

Giải thích vai trò của từng thư mục và file chính trong dự án — mạng xã hội du lịch + đặt chuyến/HDV + chatbot RAG. Monorepo gồm backend/ (NestJS) và frontend/ (React).

NestJS 10 + PostgreSQLReact 18 + ViteRAG v2 (Agentic)
BE Backend FE Frontend DOC Tài liệu/cấu hình

1 Thư mục gốc

Monorepo: 2 ứng dụng tách biệt + tài liệu, cấu hình deploy.

DoAnTotNghiepv3/ ├─ backend/ # API server NestJS (port 8080) ├─ frontend/ # Web app React + Vite (port 3000) ├─ documemtRAG/ # Tài liệu .txt cho chatbot RAG học ├─ docs/ # Báo cáo, sơ đồ, trang HTML lý thuyết ├─ render.yaml # Cấu hình deploy backend lên Render └─ CLAUDE.md # Sổ tay dự án (nguồn sự thật cho AI agent)
Thư mục / FileTác dụng
backend/Toàn bộ API: xác thực, chuyến đi, HDV, ví, chatbot… Chạy độc lập với frontend.
frontend/Giao diện người dùng (web + đóng gói Android qua Capacitor). Gọi API của backend.
documemtRAG/Kho tài liệu du lịch dạng văn bản. Lệnh rag:ingest đọc các file này, cắt nhỏ rồi tạo vector cho chatbot.
docs/Tài liệu báo cáo, sơ đồ kiến trúc (.drawio), và các trang HTML cơ sở lý thuyết.
render.yamlBlueprint deploy: khai báo web service + PostgreSQL + biến môi trường cho Render.
CLAUDE.mdGhi chú toàn bộ kiến trúc, quy ước, trạng thái tính năng — đọc trước khi sửa code.

2 backend/src BE

Gốc mã nguồn backend: điểm khởi động, module gốc, cấu hình DB.

File / Thư mụcTác dụng
main.tsĐiểm khởi động ứng dụng: tạo app NestJS, bật CORS, mở Swagger tại /docs, lắng nghe cổng 8080.
app.module.tsModule gốc — gom tất cả module nghiệp vụ + cấu hình toàn cục (TypeORM, scheduler, interceptor, guard).
datasource.tsCấu hình TypeORM cho CLI (chạy migration độc lập với app).
database/Hàm dựng cấu hình kết nối PostgreSQL từ biến môi trường (dùng chung cho app + CLI).
common/Thành phần dùng chung mọi module: decorator, guard, filter, interceptor, DTO, enum.
data/Dữ liệu tĩnh — kho kiến thức 63 tỉnh thành cho trợ lý AI tra cứu.
modules/Toàn bộ nghiệp vụ, mỗi tính năng là một module riêng (xem mục 4).
rag-ingest.script.tsScript nạp tài liệu vào vector store: đọc documemtRAG/ → cắt chunk → tạo embedding → lưu DB.
seed-curated-trips.tsScript tạo sẵn 20 chuyến đi mẫu (idempotent theo tiêu đề) để demo.

3 common — thành phần dùng chung BE

Code tái sử dụng xuyên suốt mọi module, đặt một chỗ để nhất quán.

Thư mụcTác dụng
decorators/@Public() (bỏ qua xác thực), @CurrentUser() (lấy user từ token), @Roles() (phân quyền).
guards/JwtAuthGuard (bắt buộc đăng nhập), RolesGuard (chặn theo vai trò), OptionalJwtAuthGuard (đăng nhập tuỳ chọn).
filters/GlobalExceptionFilter — chuẩn hoá mọi lỗi trả về cùng một định dạng.
interceptors/ResponseInterceptor — bọc mọi response vào { data, message, success, statusCode }.
dto/DTO dùng chung (vd phân trang).
enums/Hằng liệt kê dùng nhiều nơi, vd vai trò người dùng (UserRole).
types/Kiểu TypeScript dùng chung, vd payload JWT.

4 modules — nghiệp vụ BE

Mỗi tính năng = 1 module, thường gồm: *.controller.ts (route HTTP), *.service.ts (logic), entities/ (bảng DB), dto/ (kiểm tra dữ liệu vào), *.module.ts (lắp ghép).

ModuleTác dụng
auth/Đăng ký / đăng nhập, JWT access + refresh, xác thực email bằng OTP 6 số.
user/Hồ sơ người dùng, theo dõi (follow), sở thích du lịch để cá nhân hoá gợi ý.
trip/Chuyến đi: tạo/sửa, lịch trình, thành viên, yêu cầu tham gia, gợi ý (recommend), huỷ chuyến.
guide/Hướng dẫn viên: hồ sơ, đặt tour (booking), ví tiền, giao dịch, rút tiền, lịch sử tour.
place/Địa điểm du lịch + danh mục + tỉnh thành; admin quản lý CRUD.
post/Bài viết mạng xã hội: đăng, thích, bình luận, feed.
review/Đánh giá đa hình (địa điểm / chuyến / HDV / thành viên) — tính lại rating thật.
message/Nhắn tin: hội thoại 1-1 + nhóm (chat nhóm chuyến đi) qua WebSocket + REST.
notification/Thông báo đẩy + đánh dấu đã đọc.
payment/Nạp tiền qua SePay (tạo QR + webhook xác nhận).
saved/Lưu (bookmark) chuyến / HDV / bài viết.
upload/Tải ảnh: ưu tiên Firebase Storage, không có thì lưu local uploads/.
mail/Gửi email OTP (Nodemailer); chưa cấu hình SMTP thì in mã ra console.
admin/Bảng điều khiển quản trị: thống kê, khoá user, duyệt HDV, nạp tiền hàng loạt.
ai/Trợ lý AI v1 (pipeline 3 bước HIỂU→LÀM→TRẢ) + tạo chuyến từ hội thoại.
ragv2/Chatbot RAG v2 — kiến trúc agent tool-calling (xem mục 5).
Mẫu một module — ví dụ trip/: trips.controller.ts nhận request → trips.service.ts xử lý nghiệp vụ + truy vấn → entities/trip.entity.ts ánh xạ bảng trips → dto/trip.dto.ts kiểm tra dữ liệu gửi lên → trips.module.ts khai báo và kết nối tất cả.

5 modules/ragv2 — chatbot RAG BE

Trái tim chatbot: agent tự gọi tool truy hồi đa nguồn (DB thật + tài liệu vector).

FileTác dụng
ragv2.controller.tsRoute công khai /rag-v2/ask|status|ingest.
ragv2.service.tsĐiều phối: nạp tài liệu (ingest), chọn provider LLM, rẽ nhánh agent / pipeline.
ragv2.agent.tsVòng lặp ReAct: LLM tự quyết gọi tool nào → nhận kết quả → trả lời. Có lớp chống bịa.
entities/knowledge-chunk.entity.ts — bảng rag_knowledge_chunks lưu nội dung + vector.
lib/rag-tools.ts6 tool agent: tìm trip/place/guide/post, tra tài liệu, dựng lộ trình.
lib/retrievers/Mỗi nguồn DB một retriever (trip/place/guide/post) — query trực tiếp bằng SQL.
lib/chunker.tsCắt tài liệu thành đoạn theo câu trọn vẹn (có gối đầu).
lib/gemini-* / openai-*2 nhà cung cấp embedding + chat (Gemini / OpenAI), đổi qua biến môi trường.
lib/cosine.ts · fusion.ts · lexical.tsToán truy hồi: cosine (dense), RRF (hợp nhất), BM25 + tokenizer tiếng Việt (sparse).

FileTác dụng

7 frontend/src FE

Mã nguồn web app: trang, component, gọi API, state, kiểu dữ liệu.

File / Thư mụcTác dụng
main.tsxĐiểm khởi động React: gắn App vào DOM, bọc các provider (React Query…).
App.tsxComponent gốc — dựng router và bố cục chung.
routes/Khai báo router (createBrowserRouter): nhóm layout chính / auth / admin + ProtectedRoute.
pages/Mỗi màn hình là một trang (xem mục 8).
components/Thành phần UI tái dùng: primitive, layout, theo domain (mục 9).
layouts/Khung trang: MainLayout (có nav), AuthLayout, AdminLayout.
services/Mỗi domain một file gọi axiosInstance → bóc dữ liệu → chuyển sang kiểu FE. Có axiosInstance.ts tự gắn token + tự refresh.
hooks/Wrapper React Query cho mỗi domain: useTrips, useGuides, useAdmin…
store/State client bằng Zustand: authStore, currentUserStore, aiAssistantStore…
types/Định nghĩa interface domain (User, Trip, Place, Post…).
constants/Hằng số: đường dẫn route, cấu hình app, bộ lọc.
utils/Hàm tiện ích: cn() gộp class, format ngày, tính trạng thái chuyến.
styles/CSS toàn cục + cấu hình Tailwind.

Mỗi thư mục là một trang: <Name>Page.tsx + index.ts re-export, đôi khi kèm components/ / tabs/ riêng.

TrangTác dụng

Guides · GuideDetail · GuideApply · GuideDashboardDanh sách HDV, hồ sơ HDV (giới thiệu / lịch / lịch sử tour / đánh giá), đăng ký làm HDV, bảng điều khiển HDV.
PlacesKhám phá địa điểm + chi tiết.
SocialMạng xã hội: feed bài viết.
MyBookings · BookingDetailLịch sử đặt tour của khách + chi tiết.

Profile · EditProfile · UserProfileHồ sơ cá nhân, chỉnh sửa, xem hồ sơ người khác.

10 docs & cấu hình DOC

Tài liệu, sơ đồ và các file cấu hình quan trọng.

File / Thư mụcTác dụng
docs/Báo cáo + trang HTML lý thuyết (chatbot, recommender, tiền xử lý RAG, cấu trúc dự án) + sơ đồ .drawio.

frontend/.env.*Cấu hình theo môi trường (dev / production / lan): chủ yếu VITE_API_BASE_URL trỏ tới backend.

Quy ước chung: Backend tổ chức theo module nghiệp vụ (controller → service → entity → dto). Frontend tách page (màn hình) và service/hook (gọi API) — không để logic gọi API lẫn trong giao diện. Mọi response API đều có dạng thống nhất { data, message, success, statusCode }.
